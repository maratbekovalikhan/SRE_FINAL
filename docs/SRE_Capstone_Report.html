1. Executive Summary

This project prepares a FastAPI Task Manager API for a Production Readiness Review with Terraform, Docker, Kubernetes, Prometheus, Grafana, Alertmanager, GitHub Actions, and load testing.

2. Architecture

  • Application layer: FastAPI CRUD API with PostgreSQL and Redis
  • Platform layer: Kubernetes deployments, services, ingress, and HPA
  • Observability layer: Prometheus, Grafana, Alertmanager
  • Delivery layer: GitHub Actions CI/CD and GHCR

3. Infrastructure as Code

Terraform is organized into a local environment, an optional AWS scaffold, reusable modules, and a bootstrap directory for S3 + DynamoDB remote state.

terraform/
├── bootstrap/
├── environments/
│   ├── local/

4. CI/CD

The CI workflow runs linting, tests, Terraform validation, Kubernetes schema checks, and Docker build validation. The CD workflow builds and pushes a GHCR image, runs a Trivy scan, and deploys to a cluster when KUBE_CONFIG_DATA is configured.

5. Observability

The service exports HTTP, task-operation, cache, and synthetic workload metrics. Prometheus scrapes /metrics. Grafana provisions a dashboard with availability, latency, throughput, cache, and business-operation panels. Alert rules cover error rate, latency, task write failures, pod restarts, and node pressure.

6. SLOs

SLI SLO Alert
Availability >= 99.9% HighErrorRate
p99 latency < 500ms HighLatency
Task write success ratio >= 99% TaskWriteFailureRate

7. Scaling

The HPA targets 70% CPU with a minimum of 2 replicas and a maximum of 10 replicas. The /work endpoint supports cpu_iterations so load tests can trigger realistic CPU pressure for a live demo.
